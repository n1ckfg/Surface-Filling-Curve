# Architecture

This document outlines the architecture of the Surface-Filling Curve generator, which implements a pipeline for generating field-aligned surface-filling curves on 3D meshes.

## High-Level Overview

The system is composed of a C++ core executable that performs the heavy lifting and geometric processing, alongside a Python script (`__init__.py`) that wraps this executable as a Blender add-on for a user-friendly interface.

### Data Flow

The core processing pipeline follows a linear data flow:
1. **Input Mesh**: The system loads a 3D mesh (via `.ply` or `.obj` files).
2. **Field Optimization**: Generation of a direction field and a scalar field representing parallel stripes.
3. **Implicit Stitching**: Modifying the scalar field's topology to connect disparate stripes.
4. **Isoline Extraction**: Extracting the final curve geometry from the modified scalar field.
5. **Output Curve**: The resulting curve, optionally post-processed, is saved or imported back into Blender.

## Core Modules

The C++ core is highly modular, with specific responsibilities divided into distinct classes and files.

### 1. Orchestration (`src/curve.h`, `src/curve.cpp`)
The `Curve` class acts as the central coordinator (facade) for the entire process. It manages the sequence of operations, instantiates the necessary sub-modules, and handles data flow between them.

### 2. Field Generation (`src/stripe.h`, `src/stripe.cpp`)
The `Stripe` module is responsible for computing the foundational fields required for curve generation:
- It optimizes a direction field on the mesh.
- It generates a scalar field whose isolines represent parallel stripes over the surface.
- It utilizes `MeshHierarchy` (`src/hierarchy.h`), a multi-resolution data structure (inspired by approaches like Instant Meshes) to perform these optimizations efficiently.

### 3. Implicit Stitching (`src/stitcher.h`, `src/stitcher.cpp`)
The `Stitcher` represents the core algorithmic contribution of this project. The initial stripes generated by the `Stripe` module typically form many disconnected closed loops. The `Stitcher` modifies the underlying scalar field to conceptually "break" and "reconnect" these loops, ensuring that the resulting isolines form a single, continuous, non-intersecting surface-filling curve.

### 4. Isoline Extraction (`src/isolines.h`, `src/isolines.cpp`)
Once the scalar field has been stitched, the `Isolines` module extracts the explicit geometry of the curve (a polyline). It also provides post-processing capabilities, such as "repulsion" to improve the uniformity of the spacing between adjacent segments of the curve.

### 5. Utilities & Loaders
The project implements minimal, custom utilities to minimize external dependencies:
- **Loaders**: `PlyLoader` (`src/plyloader.cpp`) and `ObjLoader` (`src/objloader.cpp`) handle importing mesh geometry.
- **Parallelization**: The `Parallel` module provides wrappers for multi-threaded execution.
- **Math**: Basic linear algebra and mathematical operations are encapsulated in `MathUtils`.

## Blender Integration (`__init__.py`)

The Blender add-on provides a GUI within Blender to use the tool. It acts as an orchestrator for the C++ CLI:
- It exports the selected Blender mesh to a temporary file.
- It invokes the compiled C++ executable as a subprocess, passing user-configured parameters.
- Once the executable finishes, the add-on imports the resulting curve data back into the Blender scene as a native curve object.
